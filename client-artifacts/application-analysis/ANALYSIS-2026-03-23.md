## Executive Summary

Six independent deep analyses of the SafeTrekr codebase converge on a clear picture: **the platform has strong conceptual bones -- SHA-256 hash-chain evidence logging, 120+ database CHECK constraints, a well-structured module boundary between three portals, and a genuinely novel positioning in group travel safety -- but it has critical security and compliance gaps that are incompatible with a platform managing sensitive data about minors.**

### The 5 Most Important Findings (Cross-Agent Consensus)

| # | Finding | Flagged By | Severity |
|---|---------|-----------|----------|
| 1 | **Row Level Security is effectively non-functional.** 44 of 137 tables have RLS disabled (many with dead policies). The remaining tables are bypassed by the service-role client used in 90.6% of backend queries. The `anon` role can TRUNCATE `evidence_logs` and DELETE all `medical_information`. | Backend, Security, Database | CRITICAL |
| 2 | **COPPA/FERPA non-compliance with no path to remediation.** No verifiable parental consent, no read-access audit logging, no data retention/purge, no data processing agreements with schools. Platform processes K-12 minors' location, medical, and passport data without legal basis. | Security, Database, Product Strategy | CRITICAL |
| 3 | **Production secrets exposed.** `stripe_creds.md` contains plaintext Stripe dashboard credentials at project root. `SECRET_KEY` is a test value. SendGrid webhook signature verification is disabled. JWT audience verification is off. | Security | CRITICAL |
| 4 | **The `finalize_trip` pipeline (950 lines, 15+ sequential HTTP calls) has no transaction or rollback.** A failure at step 7 leaves trips in an irrecoverable inconsistent state with orphaned participants, flights, and lodging but no venues, itinerary, or invites. | Backend, Database | CRITICAL |
| 5 | **The frontend is a client-rendered SPA masquerading as a Next.js app.** 113 of 119 pages are `'use client'`. Zero server components, zero `loading.tsx`/`error.tsx`, zero code splitting for AG-Grid (500KB+) and MapLibre (300KB+), no `middleware.ts` for auth, no ESLint, and a testing score of 1/10. | React Developer, UX Designer | HIGH |

**The bottom line**: The immediate priority is security and compliance remediation (Findings 1-3), followed by data integrity (Finding 4), then frontend architecture (Finding 5). Revenue-generating product work (evidence binder viewer, per-traveler pricing display, self-service signup) should proceed in parallel on a separate track from the security work.

## Architecture Recommendations

Synthesized from Backend, React Developer, and Database findings into a coherent architecture evolution plan.

### Current Architecture

```
[Next.js Frontend (SPA)]  --Supabase JS--> [Supabase PostgreSQL]
         |                                       ^
    [Supabase Auth]                              |
         |                                  [PostgREST HTTP]
    [JWT Token]                                  |
         |                              [FastAPI Core API]
         +------Core API calls----------> port 8001
                                              |
                               +------+------+------+------+
                               |      |      |      |      |
                            [Redis] [SendGrid] [Stripe] [TarvaRI]
                            (sync!)  (no CB)  (no CB)  (30s timeout)
```

### Target Architecture (3-Month Horizon)

```
[Next.js Frontend (Hybrid SSR/CSR)]
  |-- middleware.ts (Supabase SSR auth, route protection)
  |-- Server Component shells (data fetch + HydrationBoundary)
  |-- Client Islands (interactive forms, grids, maps)
  |-- Dynamic imports (AG-Grid, MapLibre -- lazy loaded)
  |
  +--[Supabase JS Client]--> [Supabase PostgreSQL]
  |                           |-- RLS enabled on ALL tables
  +--[Core API Client]------> |-- JWT-claim-based policies
                               |-- Composite tenant FKs
                          [FastAPI Core API]
                            |-- Async Redis (redis.asyncio)
                            |-- Shared httpx.AsyncClient (lifespan)
                            |-- Circuit breakers (TarvaRI, SendGrid, etc.)
                            |-- User-scoped Supabase client (default)
                            |-- Service-role (allowlisted flows only)
                            |-- Finalize -> Supabase RPC (single transaction)
                            |-- Response models on all endpoints
                            |-- Outbox worker queue (email, push, AI)
```

### Key Architecture Decisions

| Decision | Alternatives Considered | Rationale | Revisit If |
|----------|------------------------|-----------|------------|
| **Keep monolith, improve boundaries** | Microservices | All problems are boundary/consistency within one app. Microservices add network complexity without solving the actual issues. (GPT + Gemini consensus) | Team grows beyond 5 backend engineers |
| **Supabase RPC for finalize** | Saga orchestrator, manual compensation | Only reliable way to achieve atomicity over PostgREST without infrastructure changes. Single network round-trip. | Self-hosted Postgres with direct connections |
| **Hybrid SSR/CSR (not full RSC rewrite)** | Full RSC rewrite, stay SPA | Preserves existing TanStack Query mutation infrastructure while gaining server-rendered shells, streaming, and auth. | Next.js changes App Router patterns significantly |
| **JWT-claim-based RLS over user-table subqueries** | Keep subquery-based RLS | Eliminates per-query DB lookup in RLS evaluation. Scales to 10K+ rows without measurable overhead. | Custom claims become too complex |
| **User-scoped Supabase as default** | Keep service-role default | Service-role bypasses ALL RLS. Must be restricted to explicitly allowlisted system flows only. | New system flows requiring cross-tenant access |
| **Vertical slice decomposition for trips.py** | Layered architecture | Each domain (participants, safety, finalize) gets its own file with all layers. Reduces merge conflicts and cognitive load. | Domain boundaries shift significantly |
