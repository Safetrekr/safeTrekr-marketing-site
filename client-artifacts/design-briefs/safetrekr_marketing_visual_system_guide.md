# Safetrekr Marketing Visual System + Color Application Guide

**Version:** 1.0  
**Date:** 2026-03-24  
**Project:** Safetrekr Marketing Site - V3
**Purpose:** Establish a shared visual language and color application system for the Safetrekr marketing website so design and engineering can build a calm, premium, high-trust public brand experience.

---

## 1. Why this document exists

Safetrekr sits in a sensitive category. It must communicate:

- trust
- preparedness
- oversight
- documentation
- visibility
- calm operational competence

That means the marketing site cannot look like:

- generic SaaS
- fear-based cybersecurity branding
- travel-lifestyle marketing
- emergency-alert software
- soft wellness branding

This guide defines how color, surfaces, hierarchy, and visual emphasis should work together so Safetrekr feels like a **modern, premium oversight platform**.

---

## 2. Visual thesis

Safetrekr should feel like:

**calm authority**  
**visible readiness**  
**modern operational elegance**  
**human-guided oversight**  
**documented accountability**

The site should create reactions like:

- “This is serious.”
- “This feels organized.”
- “This looks current.”
- “These people know what they are doing.”
- “I would trust this for real-world travel responsibility.”

---

## 3. Core design principles

### 3.1 Clarity before flourish
The first job of the site is understanding. Color and motion should sharpen comprehension, not decorate weak messaging.

### 3.2 Calm before urgency
Safetrekr deals with risk, but the brand should project preparedness—not panic.

### 3.3 Brand is not status
The green brand system is not the same thing as safety-state signaling. Brand green should communicate identity and confidence; status colors should communicate live meaning.

### 3.4 Light theme, not weak theme
The interface should feel bright and open, but still grounded, structured, and credible.

### 3.5 Premium through restraint
A premium experience comes from spacing, hierarchy, contrast, composition, and consistency—not over-decoration.

## 5. Color behavior rules

## 5.1 The neutral system builds trust
The marketing site should primarily feel built from the neutral system:

- `background` for page canvas
- `card` for contained surfaces
- `foreground` for authority text
- `border` for structure
- `muted-foreground` for support language

This creates the baseline feeling of composure and credibility.

## 5.2 Brand green expresses identity, not live status by default
Use the primary scale to express:

- brand confidence
- section accents
- call-to-action energy
- positive tone
- visual cohesion

Do **not** use brand green as a substitute for live safety-state communication unless the meaning is explicitly brand-oriented and not operationally ambiguous.

## 5.3 Secondary blue provides authority and anchor
`secondary` should be used as the “executive authority” accent. It is ideal for:

- high-trust anchors
- dense information panels
- footer areas
- contrast moments
- professional navigation states
- structured product framing

## 5.4 Safety colors must remain rare and meaningful
The safety palette should be reserved for:

- true state communication
- map markers
- alert badges
- trip status indicators
- documentation severity markers

These colors should **not** become decorative accents throughout the marketing site.

## 5.5 Warning colors are for escalation, not ambiance
Use warning tones only where a user should interpret elevated attention or caution. They should never become a recurring decorative theme.

---

## 6. Marketing-site visual hierarchy by color

## 6.1 Primary reading order
The user should first perceive:

1. strong foreground text
2. controlled neutral surfaces
3. selective brand green emphasis
4. carefully placed authority blue anchors
5. only then state colors, if relevant

## 6.2 Contrast philosophy
Contrast should come primarily from:

- light vs deep surface structure
- dark type on pale backgrounds
- selective green emphasis against quiet neutrals
- occasional dark blue grounding sections

Avoid contrast that depends on loud saturation.

---

## 7. Surface system

## 7.1 Page canvas
**Default page background:** `#e7ecee`

This is the atmospheric base. It should feel like a bright, structured, high-trust environment—not a flat white template.

Use it for:

- main page background
- spacious content fields
- large open editorial sections
- map or route motifs behind content

## 7.2 Primary content surfaces
**Card background:** `#f7f8f8`

Use for:

- product cards
- feature modules
- trust panels
- form blocks
- documentation previews
- stats blocks

These should feel lightly elevated, crisp, and premium.

## 7.3 Elevated or anchored surfaces
Use darker surfaces selectively, typically with `secondary` or a deep brand tone, for:

- footer
- high-importance CTA bands
- standout proof modules
- dense trust sections
- selected product showcases

These darker moments should be occasional and intentional.

## 7.4 Borders and dividers
**Border token:** `#b8c3c7`

Use for:

- card outlines
- section separators
- table boundaries
- chips
- field structure

Do not overuse heavy borders. Borders should support order quietly.

---

## 8. Text color roles

## 8.1 Primary text
**foreground** `#061a23`

Use for:

- hero headlines
- body text
- section headings
- high-value labels
- trust copy

This color is one of the most important trust-building assets in the system.

## 8.2 Secondary text
**muted-foreground** `#616567`

Use for:

- support copy
- captions
- eyebrow text
- metadata
- low-emphasis explanatory text

Avoid using muted text for critical messaging.

## 8.3 Text on darker surfaces
Use light text carefully on `secondary`, deep brand tones, or dense dark panels. Maintain strong readability and avoid lower-contrast muted treatments inside dark blocks.

---

## 9. Brand color application

## 9.1 Primary green scale: intended use

### Light tints (`primary-50` to `primary-200`)
Use for:

- subtle background washes
- light feature highlights
- positive framing areas
- soft CTA section accents
- low-intensity visual grouping

### Mid tones (`primary-300` to `primary-500`)
Use for:

- primary buttons
- graphic accents
- selected icons
- key emphasis lines
- active nav accents
- visual signal moments

### Deep tones (`primary-600` to `primary-950`)
Use for:

- hover states
- stronger CTA emphasis
- dark brand panels
- premium depth accents
- typography on very light brand backgrounds where contrast is required

## 9.2 Green should feel like “readiness” and “confidence”
The brand green family should visually suggest:

- preparedness
- steadiness
- oversight with humanity
- operational calm

Avoid letting the green feel:

- too healthcare-generic
- too eco-brand coded
- too soft or pastel-only
- interchangeable with live “all clear” status states

## 10. Secondary color application

**secondary** `#123646`

This color should carry:

- seriousness
- authority
- structure
- institutional confidence
- executive calm

Best uses:

- footer background
- navigation contrast elements
- data callout sections
- trust bars
- premium CTA support blocks
- product showcase framing
- dark chips or tags

This color should make the brand feel established and reliable.

## 11. Status and alert color rules

## 11.1 Safety green
**safety-green** `#22c55e`

Use only when meaning is explicitly status-oriented:

- safe trip state
- all-clear indicators
- resolved markers
- map status dots

Do not use this as the site’s general accent green.

## 11.2 Safety yellow
**safety-yellow** `#eab308`

Use for:

- caution markers
- flagged items
- elevated awareness indicators
- non-critical issue states

Should remain limited and clearly functional.

## 11.3 Safety red
**safety-red** `#ef4444`

Use for:

- alert states
- severe risk markers
- urgent incident indicators
- destructive status only

Never use red as a decorative marketing highlight.

## 11.4 Warning scale
Use warning tones for tiered caution in:

- maps
- documentation severity markers
- operational cards
- filtered status systems

Keep warning styling data-oriented and precise—not dramatic.

## 12. Component-level application

## 12.1 Hero area
Recommended treatment:

- page background on `background`
- strong `foreground` headline
- support text on `muted-foreground`
- primary CTA in `primary-500` or `primary-600`
- secondary CTA outlined with `border` and `foreground`
- supporting product composition on `card`
- optional anchor accents using `secondary`

The hero should feel bright, controlled, and high-trust.

## 12.2 Buttons

### Primary button
- Fill: `primary-500`
- Hover: `primary-600`
- Text: light for strong contrast

### Secondary button
- Surface: `card` or transparent
- Border: `border`
- Text: `foreground`
- Hover: subtle fill using `primary-50` or neutral darkening

### Tertiary / text button
- Text: `foreground` or `secondary`
- Underline or subtle icon support optional

Avoid multiple loud button colors on the same screen.

## 12.3 Cards
Default card treatment:

- Fill: `card`
- Border: `border`
- Title: `foreground`
- Support text: `muted-foreground`
- Accent: small use of brand green or deep blue only where helpful

Cards should look organized, not soft or bubbly.

## 12.4 Trust strips and proof modules
These can use:

- quiet light neutral surface
- subtle brand tint backgrounds
- occasional `secondary` anchored variants

Use color to reinforce confidence and proof, not excitement.

## 12.5 Forms
Recommended form treatment:

- Field surface: `card`
- Border: `border`
- Placeholder or low-emphasis support: `input` / `muted-foreground`
- Focus: `ring`

The focus ring color is important; it should feel visible, controlled, and professional.

## 12.6 Chips / tags
Use chips in three families:

### Brand tags
- background: `primary-50` to `primary-100`
- text: deeper primary or `foreground`

### Neutral tags
- background: `card` or `muted`
- border: `border`
- text: `muted-foreground`

### Status tags
Use only for explicit operational meaning with safety or warning colors.

## 12.7 Alerts
Marketing-site alerts should be rare.

If used:

- informational: deep neutral or blue framing
- caution: warning scale
- destructive: `destructive` only when meaning is explicit

Avoid ambient “alert UI” in brand storytelling sections.

## 12.8 Tables and documentation previews
Use:

- `card` surfaces
- `border` rules
- `foreground` for headers
- `muted-foreground` for metadata
- brand green sparingly for progress or approval cues
- warning/status colors only for actual states

Documentation previews are a differentiator. They should feel polished and evidence-grade.

## 12.9 Maps
Maps are one of Safetrekr’s signature visual opportunities.

Recommended map palette behavior:

- neutral background surfaces
- route emphasis using brand green or authority blue where appropriate
- status dots using true safety colors
- caution overlays using warning scale
- border lines subtle, not heavy

Maps should feel legible and calm—not tactical or militarized.

## 13. Section-level color recommendations

## 13.1 Open editorial sections
Use mostly neutrals with deep text and selective green accents.

## 13.2 Product showcase sections
Use `card` surfaces with occasional darker `secondary` framing to create importance.

## 13.3 CTA bands
Recommended options:

- light brand wash using `primary-50` / `primary-100`
- deeper green block using `primary-700` or similar
- occasional `secondary` anchored CTA for high-trust seriousness

## 13.4 Footer
Best candidate for `secondary` or a deep brand tone.

The footer should feel grounded, credible, and complete.

---

## 14. Accessibility and contrast guardrails

## 14.1 General rule
Elegance cannot come at the expense of readability.

## 14.2 Do

- keep primary copy in `foreground`
- use muted text only for secondary information
- test button and link contrast carefully
- ensure `ring` is clearly visible on all interactive fields
- reserve light green tints for backgrounds, not body text

## 14.3 Do not

- place muted text on light brand tints without testing contrast
- use `primary-300` or lighter greens for essential text
- rely on color alone to communicate state
- overuse yellow on pale backgrounds without sufficient contrast support

---

## 15. Visual do / don’t

## 15.1 Do

- build the site mostly from neutrals
- let green act as identity and selective emphasis
- use blue to add authority and grounding
- keep state colors operational, not decorative
- make surfaces crisp and controlled
- make documentation and map moments visually strong

## 15.2 Don’t

- turn the site into a green-washed product
- use status green as the primary brand accent everywhere
- use red or yellow as visual excitement devices
- make borders too dark or heavy
- allow multiple saturated accents to compete on one screen
- create ambiguity between brand meaning and safety-state meaning

- highlighted positive modules: `primary-100` to `primary-200`

---

## 18. Quick implementation checklist

Before approving any page or component, confirm:

- Is the neutral system doing most of the work?
- Is green being used as brand rather than accidental status?
- Are status colors limited to explicit meaning?
- Is the page still readable if accents are removed?
- Does the darker blue appear where authority or grounding is needed?
- Do documentation and map states feel precise, not dramatic?
- Does the design feel calm and premium rather than urgent or noisy?

---

## 19. One-sentence final guidance

**Safetrekr’s marketing color system should make the brand feel bright, composed, trustworthy, and operationally serious—using neutrals for structure, green for identity, blue for authority, and status colors only when meaning must be explicit.**
